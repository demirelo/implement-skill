Why it works

Objective oracle
Architects write the tests; Builders make them green.
Best-of-N
Candidates compete; smallest green diff wins.
Lens-diverse review
Each Architect reviews a different way.
Secret hygiene
Models never see keys; outbound is scrubbed.
Gates first
No review token spent on a red diff.
Worktree isolation
Each slice runs off a green baseline.
Kill criteria
Iteration caps + stop-and-ask on blockers.
No auto-merge
The PR is always left for a human.

What's built

Feature-complete end-to-end: stored onboarding with a Venice e2ee privacy lane; the Architect phases (intent · plan-consensus · acceptance-test oracle · lens-diverse review); the GitHub draft-PR + tiered handoff; guardrails (Seatbelt/Docker sandbox, destructive-command gating, worktree isolation, kill/stop-and-ask, suitability — sandbox live-verified); and a learned router (benchmark priors + this machine's local win-rates pick Builders per task, self-improving) with a KB assembler. Oracle-grounded and locally sovereign.
