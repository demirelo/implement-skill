# Performance pass — /implement hot loop (2026-07-07)

Five optimizations to the best-of-N inner loop. Wall-clock and cost are dominated by LLM dispatches
and gate runs, so these are orchestration-shape changes, not micro-tuning. All TDD'd; the pass was
adversarially reviewed (4 lenses × independent verify) before landing.

## Shipped

1. **Parallel best-of-N** (`execute.run_best_of_n`) — candidates run concurrently in a
   `ThreadPoolExecutor` instead of a serial `for`. Each competes in its own isolated `_copy_repo`,
   created + torn down **inside its own thread**, so workspace setup parallelizes with the loop and
   wall-clock ≈ the slowest single candidate (~N× on the default best_of_n=3). Winner selection
   (`min(green, key=_diff_size)`, dispatchers-order tie-break) stays deterministic regardless of
   completion order.

2. **~~Git worktrees~~ → parallel copytree** — *the worktree fast-path was tried and REJECTED* by the
   adversarial review (three confirmed findings): (a) creating the first candidate's `.worktrees/`
   dirties the parent, so later candidates fall back to copytree → **non-identical oracle trees within
   one run**; (b) a git worktree checks out HEAD only, so gitignored runtime files (`.env`, local
   config, golden fixtures) are **absent**, forking the oracle and wasting/failing runs; (c) a
   worktree's git-dir lives outside the sandbox workdir, so git-writing tests **fail under
   seatbelt/Docker**. Resolution: always `_copy_repo` (uniform, includes gitignored non-heavy files,
   self-contained `.git` works sandboxed), and recover the setup cost by copying **inside** the
   parallel section (independent copies, no shared git lock).

3. **Repo context once per loop** (`execute._repo_context` hoisted) — failed turns fully revert, so
   the repo is byte-identical every turn; walk + scrub it once and pass `repo_ctx` into
   `_build_prompt` as a stable prefix (provider prompt-caching can hit). `_repo_context` also
   early-stops once the char budget is full instead of reading the whole tree to truncate.

4. **Two-tier gate** (`gate.run_gate(only=...)` + `execute.run_inner_loop`) — turn 0 runs the FULL
   suite to establish the failing set; iteration turns run ONLY the failing set (scoped via adapter
   `test_one`); a scoped-green turn then runs the FULL suite to confirm before declaring success.
   **Oracle invariant preserved**: success is declared only after a full-suite pass with
   `passing_count > 0` (H5); a diff that fixes the target but breaks other tests is caught by the
   confirm and rejected. Scoped node ids that look like flags are dropped (no arg-injection).

5. **Parallel preflight** (`preflight.readiness`) — each model's credential resolves in parallel
   (`ThreadPoolExecutor.map`, order-preserving) instead of serial `op read`/probe round-trips.

## Known cosmetic (verified NOT a defect)

Two-tier `green_delta` re-bases to 0 after a regression retarget, so the first scoped-red turn after a
regression can log a one-off phantom delta. Verified worst case: a one-turn-late kill (the phantom
ages out of the 3-turn GUTTER window next turn). Oracle integrity intact; left as-is.

## Not touched

`workspace.py` (the M4 worktree-isolation primitive) remains a tested utility, not wired into the
loop — candidate isolation has always been `_copy_repo`, which meets the "never touch the operator's
live tree" guarantee on its own.
